# Cat Cipher 암호화 시스템 설계 문서 v2

## 개요

Cat Cipher는 스티커 메모장 프로그램의 암호화 기능이다.

일반적인 암호화 시스템처럼 Base64 문자열이나 무작위 영문/숫자를 표시하지 않고, 암호화된 데이터를 고양이 울음소리 형태로 변환하여 사용자에게 보여준다.

사용자는 암호화된 메모를 볼 때 실제 암호문이 아니라 "고양이 언어"를 보는 경험을 하게 된다.

---

## 목표

* AES-GCM 기반 실제 암호화 제공
* 한글, 영어, 숫자, 특수문자, 이모지 지원
* 화면에는 고양이 울음소리만 출력
* Base64, Hex 문자열 노출 금지
* 구분자 사용 금지
* 시작/종료 문구 사용 금지
* 저장 데이터와 표시 데이터 동일
* 자동 줄바꿈 데이터 삽입 금지

---

## 암호화 구조

```text
평문
↓
UTF-8 인코딩
↓
AES-256-GCM 암호화
↓
암호문(Byte Array)
↓
Cat Cipher 인코딩
↓
고양이 문자열
```

---

## 복호화 구조

```text
고양이 문자열
↓
Cat Cipher 디코딩
↓
암호문(Byte Array)
↓
AES-256-GCM 복호화
↓
UTF-8 디코딩
↓
평문
```

## Cat Cipher 사전

### 4비트(16진수) 기반 인코딩

각 토큰은 4비트를 표현한다.

| 비트   | 토큰  |
| ---- | --- |
| 0000 | 냐   |
| 0001 | 냥   |
| 0010 | 냐옹  |
| 0011 | 야옹  |
| 0100 | 냐아  |
| 0101 | 냐앙  |
| 0110 | 냐웅  |
| 0111 | 냥냐  |
| 1000 | 냐냐  |
| 1001 | 냥냥  |
| 1010 | 냐오  |
| 1011 | 냐옹냐 |
| 1100 | 먀   |
| 1101 | 먀옹  |
| 1110 | 미야  |
| 1111 | 미야옹 |

---

## 인코딩 방식

1 Byte = 8 Bit

```text
8 Bit
↓
4 Bit + 4 Bit
↓
토큰 2개
```

예시

```text
Byte
11101101

분리
1110 1101

변환
미야 + 먀옹

결과
미야먀옹
```

## 출력 예시

암호화 전

## 토큰 파싱 규칙

구분자는 사용하지 않는다.

복호화 시 가장 긴 토큰부터 우선 매칭한다.

예시

```text
미야옹
```

↓

```text
미야옹
```

(미야 + 옹 아님)

### 우선순위

```text

먀옹

야옹

냥냥
냥냐
냐앙

냐냐

```

토큰 길이 내림차순으로 탐색한다.

## 최종 컨셉

사용자는 "암호화" 버튼을 누르면 메모가 일반적인 암호문이 아닌 고양이 언어로 변하는 경험을 한다.

실제 내부에서는 AES-GCM으로 안전하게 암호화되며, 사용자에게는 오직 고양이 울음소리만 표시된다.

목표는 "보안"과 "고양이 컨셉"을 동시에 만족하는 것이다.
